Transaction Processing Information Systems (TPIS)

Information systems have become vital for organizations today and in some industries, even survival is difficult without extensive use of information technology.

Different types of information systems must be developed to meet the whole array of information needs in a company: systems for processing transactions, management information systems and decision support systems.

  1. Strategic level reports to provide managers with the information they need to plan activities
  2. Tactical (and status) situation reports to enable management to draw up new or revised short-term plans (from 1 to 3 years)
  3. Operational reports with daily information to keep managers fully informed of the company’s current situation

Transaction Processing Systems (TPS)

Transaction processing systems are the cornerstones of a company’s information system and compile their daily business operations. Collect, store, modify and retrieve the transactions of an organization.

The four important characteristics of a TPS are:
  • RAPID RESPONSE
  • RELIABILITY
  • INFLEXIBILITY
  • CONTROLLED PROCESSING
Types of Transaction Processing Systems
  • Batch Transaction Processing
    Examples:
    • CHEQUE CLEARANCE
    • BILL GENERATION
    • CREDIT CARD TRANSACTIONS
  • Real-Time Transaction Processing
    Two main concerns:
    • CONCURRENCY
    • ATOMICITY
    Examples:
    • RESERVATION SYSTEMS
    • POS TERMINALS
    • LIBRARY LOAN SYSTEM
Data Validation is used to check the entry of transaction data. Involves procedures to ensure transactions are correct and accurately stored in the database.
  • TRANSACTION INITIATION
  • FIELD CHECKING
Storing and retrieving

A TPS requires an efficient method for the storage and retrieval of data such as a database or a data warehouse.

A database is an organized collection of data.

Types of database:

  • Hierarchical database
  • Network database
  • Relational database
Important features in designing a database for RTTP:
  • Good data placement
  • Short transactions
  • Real-time backups
  • High normalization
  • Archiving of historical data
  • Good hardware configuration
Types of file in TPS
  • MASTER FILE
  • TRANSACTION FILE
  • REPORT FILE
  • WORK FILE
  • PROGRAM FILE

A DATA WAREHOUSE is a database that collects information from different data sources. Data is gathered in real-time transactions can be used for analysis in an efficient manner if it is stored in a data warehouse. Provides data that is:

  • CONSOLIDATED
  • SUBJECT-ORIENTATED
  • HISTORICAL
  • READ-ONLY
Backup and Recovery Procedures

A backup is another copy of the data that could be used to rebuild the system. If the system goes down, the recovery process rebuilds the system.

Recovery process

A TPS may fail for a number of reasons, such as system failures, human error, hardware failure, incorrect or invalid data, program errors, computer viruses or natural disasters.

  • BACKUP
  • JOURNAL
  • CHECKPOINT
  • RECOVERY MANAGER
  • MAGNETIC TAPE
  • GRANDFATHER-FATHER-SON
  • PARTIAL BACKUPS
Updating Data
  • Update in Batch
    Stages in batch processing:
    1. The collecting and storage of the transaction data in a transaction file. It involves sorting the data into sequential order.
    2. The processing of the data by updating the master file. This is not always a simple process. The updating of the master file may involve data additions, updates and deletions that need to happen in a certain order. If one error occurs in the batch, then the entire batch is rejected.
  • Update in Realtime

Data Analysis for a Transaction Processing System

The results of processing transaction are stored in a database and analyzed in many ways to meet the information needs of users.

Decision Support System

Which use data mining to find relationships and patterns in data stored in a database. This may uncover trends between the sales of certain things for instance such as pies and sauce.
